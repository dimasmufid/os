# Phase 2 — Deepen Identity & Emotional Connection

## Goal
Make the world **feel more alive**, deepen the user’s sense of identity, and increase emotional attachment.

---

# 1. Core Features

## 1.1 Avatar Emotion System
Hero emotions change based on:
- Streak strength
- Recent level-up
- World upgrades
- Idle time

Example states:
- Happy
- Calm
- Determined
- Energized

## 1.2 Ambient World Effects
To make the world feel alive:
- Floating particles (leaves, dust, fireflies)
- Subtle glow animations on lamps
- Cloud movements
- Animated plants or fountains (optional)

## 1.3 Task Templates Linked to Rooms
Each task is tied to a room:
- “Math Study” → Study Room
- “Build Altiera” → Build Room
- “Workout” → Training Room

Room context gives symbolic meaning.

## 1.4 Room Perks (Passive Bonuses)
- Study Room: +5% XP for learning tasks
- Build Room: +10% gold for creation tasks
- Training Room: discipline achievements

## 1.5 Simple Onboarding Storyline
Short introduction:
- Who you are
- What the world represents
- Why missions matter
- How to grow this world

Optional NPC or floating text panels.

---

# 2. Deliverables
- Emotion sprite variations
- Ambient particle overlays
- Room-perk implementation
- Task-bound-to-room system
- Onboarding screen flow

---

# 3. Success Criteria
- User feels bonding with avatar
- World feels alive, not static
- Tasks become symbolically meaningful
- Users understand the purpose of each room
